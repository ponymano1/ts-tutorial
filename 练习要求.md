# TypeScript 练习要求

本文档列出了每个章节的练习要求，请按照要求编写代码来验证自己是否掌握相关知识点。

## 01. 基础类型

### 练习要求
1. 声明一个 `string` 类型的变量，赋值为你的名字
2. 声明一个 `number` 类型的变量，赋值为你的年龄
3. 声明一个 `boolean` 类型的变量，表示是否是学生
4. 创建一个数字数组，包含 5 个元素
5. 创建一个对象，包含 `name`(string) 和 `age`(number) 两个属性
6. 声明一个 `any` 类型的变量，并赋予不同类型的值
7. 声明一个 `unknown` 类型的变量，并使用类型守卫进行类型收窄
8. 编写一个函数，接收两个 `number` 参数，返回它们的和（需要标注类型）

## 02. 接口与类型

### 练习要求
1. 定义一个 `User` 接口，包含 `id`(number)、`name`(string)、`email`(可选的string)
2. 创建一个符合 `User` 接口的对象
3. 使用 `type` 定义一个联合类型 `Status`，值为 'pending' | 'success' | 'error'
4. 定义一个类型别名 `ID`，可以是 `number` 或 `string`
5. 对比接口和类型别名的使用场景，写出它们的主要区别（注释形式）

## 03. 高级类型

### 练习要求
1. 定义一个联合类型变量，可以是 `number` 或 `string`，并赋予不同类型的值
2. 使用交叉类型（`&`）组合两个类型，创建一个同时具有两者属性的对象
3. 定义一个字面量类型 `Direction`，值只能是 'up' | 'down' | 'left' | 'right'
4. 编写一个泛型函数 `identity<T>`，接收任意类型参数并返回相同类型
5. 使用 `Partial<T>` 将一个接口的所有属性变为可选
6. 使用 `Pick<T, K>` 从接口中选择特定属性
7. 使用 `Omit<T, K>` 从接口中排除特定属性

## 04. 函数特性

### 练习要求
1. 编写一个函数，包含一个可选参数（使用 `?`）
2. 编写一个函数，包含一个带默认值的参数
3. 编写一个函数，使用剩余参数（`...args`）接收任意数量的参数
4. 为同一个函数编写多个重载签名，支持不同的参数类型组合

## 05. 对象与数组操作

### 练习要求
1. 使用展开运算符（`...`）浅拷贝一个对象
2. 使用展开运算符浅拷贝一个数组
3. 使用解构赋值和剩余属性从对象中提取部分属性

## 06. 类

### 练习要求
1. 定义一个 `Person` 类，包含 `name` 和 `age` 属性，以及构造函数
2. 为类添加 `public` 和 `private` 访问修饰符
3. 定义一个接口，然后创建一个类实现（`implements`）该接口

## 07. 枚举

### 练习要求
1. 定义一个枚举类型 `Color`，包含 Red、Green、Blue
2. 思考并写出为什么教程推荐使用联合类型代替枚举（注释形式）

## 08. 类型断言

### 练习要求
1. 使用 `as` 关键字进行类型断言
2. 使用非空断言操作符 `!`（并理解其风险）
3. 写出类型断言的使用场景和注意事项（注释形式）

## 09. JavaScript 与 TypeScript

### 练习要求
1. 编写一个箭头函数，接收两个参数并返回它们的和
2. 使用 `Array.map()` 将数字数组转换为字符串数组
3. 使用 `Array.filter()` 过滤出数组中的偶数
4. 使用 `Array.reduce()` 计算数组的总和
5. 链式调用多个数组方法（map + filter + reduce）
6. 对比箭头函数和普通函数中 `this` 的绑定行为

## 10. 异步与 Promise

### 练习要求
1. 创建一个返回 `Promise<string>` 的函数
2. 使用 `async/await` 语法编写异步函数
3. 使用 `Promise.all()` 并行执行多个 Promise，观察返回类型
4. 使用 `Promise.allSettled()` 处理可能失败的多个 Promise
5. 在 async 函数中使用 try-catch 进行错误处理

## 11. 数组陷阱

### 练习要求
1. 编写一个 `map()` 函数，故意忘记 `return`，观察返回类型变为 `void[]`
2. 使用 `filter()` 过滤数组，理解类型收窄的局限性
3. 使用 `reduce()` 时正确标注累加器和返回值类型
4. 调试一个链式数组操作，找出类型错误的原因

## 12. this 绑定与上下文

### 练习要求
1. 创建一个对象方法，将其赋值给变量后调用，观察 `this` 丢失
2. 使用 `call()` 显式绑定 `this` 上下文
3. 使用 `apply()` 传递数组参数并绑定 `this`
4. 使用 `bind()` 创建一个固定 `this` 的新函数
5. 对比箭头函数和普通函数在对象方法中的 `this` 行为
6. 在函数参数中显式标注 `this` 类型

## 13. 请求超时模式

### 练习要求
1. 编写一个简单的 Promise，使用 `setTimeout` 模拟延迟
2. 实现一个超时 Promise，在指定时间后自动 reject
3. 使用 `Promise.race()` 实现带超时的网络请求
4. 使用 `AbortController` 取消一个 fetch 请求
5. 理解为什么网络请求需要超时机制（注释形式）

## 14. 模块系统

### 练习要求
1. 创建一个模块，使用 `export` 导出多个命名导出
2. 创建一个模块，使用 `export default` 导出默认值
3. 在另一个文件中使用 `import` 导入命名导出
4. 使用 `import type` 进行类型导入
5. 理解 ES Module 和 CommonJS 的区别（注释形式）

## 15. 区块链开发 (Ethers.js)

### 练习要求
1. 创建一个 ethers.js 的 Provider 连接到区块链网络
2. 创建一个 Wallet 签名器
3. 使用 Contract 类实例化一个智能合约
4. 查询账户余额
5. 估算一个交易的 gas 费用
6. 使用 `staticCall` 模拟合约调用（不消耗 gas）
7. 使用 TypeChain 生成类型安全的合约绑定
8. 理解 TypeChain 如何提供类型安全（注释形式）

---

## 学习建议

1. **按顺序学习**：建议按照章节顺序逐步学习，前面的知识是后面的基础
2. **动手实践**：每个练习都要亲自编写代码，不要只看不写
3. **理解类型**：注意观察 TypeScript 的类型推断和类型检查
4. **查看错误**：遇到类型错误时，仔细阅读错误信息，理解问题所在
5. **参考教程**：遇到困难时，可以查看 `src/` 目录下对应章节的示例代码
6. **举一反三**：完成基础练习后，尝试变化参数和场景，加深理解

## 验证方法

1. 使用 `tsc` 命令编译你的代码，确保没有类型错误
2. 使用 `node` 运行编译后的 JavaScript 文件，验证运行结果
3. 在 VS Code 等编辑器中，观察类型提示和自动补全是否正确
